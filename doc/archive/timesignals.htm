KOB Time Signals

Time signals are broadcast on the MorseKOB system on the following wires:

Wire Schedule
11 daily at noon EST*
102 continuous
103 hourly clock set

*The daily time signals occur at noon Eastern Standard Time throughout the year. During the summer, therefore, the time signals are heard at 1:00 PM EDT.

Format

The time signals are transmitted as a three-minute sequence of pulses, each pulse being sent once per second. The pulses are 400 ms in length. The sequence of pulses concludes with a longer, one-second pulse, which starts exactly at the top of the hour (or, in the case of the continuous time signals on wire 102, at an exact five minute interval). Some of the pulses in this sequence are intentionally omitted, allowing the particular minute in the sequence to be easily identified. The missing pulses for the daily and hourly time signals are shown in the following table.[1] The same basic pattern is used for the continuous time signals.

Minute Missing pulses
57 29, 53, 56-59
58 29, 53-54, 56-59
59 29, 51-59

Hourly clock set signal

A different format is used for the hourly clock set signal. In this case a simple two-second pulse is sent at the top of each hour. The pulse starts at second hh:00:00 and ends at second hh:00:02. This pulse corresponds to the telegraph signal that Western Union used to send for automatically setting clocks made by the Self Winding Clock Company.

Accuracy

These time signals are generated by a copy of MorseKOB 3.0 running on a dedicated computer with the Ubuntu Linux operating system. The computer's system clock is continuously synchronized with time references on the Internet and is generally within 2 milliseconds of correct time. The system clock's statistics can be seen on the "AC" KOB server's information page at mtc-ac.dyndns.org/ntpstats/loopstats. The numbers in the third column show the clock's variation in seconds at various times during the current day.

A number of delays are incurred before the time signal is eventually heard by the listener. These delays are described in the following table:

Delay Estimated duration Comments
MorseKOB processing time <10 ms
Internet propagation time to KOB server <3 ms This estimate assumes the time signal program is running on the same computer as the KOB server, which is generally the case. Otherwise, an additional 50-100 ms could be incurred.
KOBServer processing time <10 ms
Internet propagation time to user's computer <100 ms This delay will vary depending on the user's location and network congestion.  A better estimate can be obtained by "pinging" the server at mtc-kob.dyndns.org and dividing the roundtrip delay by 2.
MorseKOB processing time <160 ms The MorseKOB program has a built-in delay of 150 ms. To this must be added an allowance for processing time.

Based on the above analysis, the time signal can be expected to lag behind true time by as much as 300 ms. Tests against WWV need to be performed to validate this analysis.

Break behavior

For the daily time signals, the time signal program connects to the wire at four minutes before the hour. At three minutes and five seconds before the hour, the program "breaks" the wire (opens the circuit), and starts the time signal pattern five seconds later.

If a user is transmitting on the wire when the time signal breaks in, and the user continues to transmit after the time signal pattern commences, the time signals will be corrupted. This will be apparent because there will not be any gaps where the missing pulses should occur. When the gap for a missing pulse is heard, this indicates that the time signals have resynchronized and can be relied on.

MorseKOB 2.5 speed setting

MorseKOB 2.5 has a "feature" whereby all marks and spaces are truncated to the equivalent of 10 dot lengths, based on the WPM code speed setting on the MorseKOB screen.  In normal use, this mechanism helps the receiving program catch up with the sender in case network congestion causes it to fall behind.  Unfortunately, it also has the undesirable side-effect of potentially distorting the time signals, making the pulses sound uneven.

For this reason, it's important to set the MorseKOB speed setting to 10 WPM when listening to time signals (5 WPM for the hourly clock set signal).

This limitation does not apply when listening to time signals with MorseKOB 3.0.
